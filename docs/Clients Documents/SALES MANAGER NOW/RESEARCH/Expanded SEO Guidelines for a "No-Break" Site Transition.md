# **SEO Guidelines for a "No-Break" Site Transition**

Here’s a detailed checklist for the development team. The goal is to ensure the new site retains all existing SEO value and is set up for future success.

 **I. Core Structure & Content Integrity**

* **Maintain URL Structure:**
  + **Guideline:** Minimize URLs changes unless absolutely necessary for structural reasons.
  + **If a URL must change:** Provide a comprehensive list of all URL changes to the SEO team before the changes are made. This allows for proper redirect mapping.
* **Preserve On-Page SEO Elements:**
  + **Guideline:** All current title tags, meta descriptions, H1 tags, and other header tags for every key page must be migrated exactly as they are.
  + **Guideline:** Ensure all valuable body copy and internal links within the content are preserved or augmented.
* **Technical SEO Configuration:**
  + **Guideline:** Ensure the new site generates a clean, error-free XML sitemap that includes all indexable pages.
  + **Guideline:** All existing canonical tags must be carried over to the new site to prevent duplicate content issues.

**II. Pre-Launch & Development Environment**

* **Analytics and Tracking Codes:**
  + **Guideline:** Ensure all existing analytics and tracking codes (like Google Analytics, Google Tag Manager, etc.) are implemented on the new site. All tracking should be thoroughly tested on the staging site.
* **Schema Markup (Structured Data):**
  + **Guideline:** All existing structured data (Schema.org markup) must be migrated to the new site. This is often used for rich snippets in search results (e.g., faq, testimonials) and is vital to maintain.

**III. Performance and User Experience**

* **Mobile-Friendliness and Core Web Vitals:**
  + **Guideline:** The new site must be mobile-friendly and pass Google's Core Web Vitals assessment for performance. A redesign can sometimes lead to slower page load times due to larger images or inefficient code.
  + **Guideline:** Test page speed on key page templates in the staging environment before launch.
* **Image Optimization:**
  + **Guideline:** Ensure all existing image alt text is carried over. For new images, alt text should be descriptive.
  + **Guideline:** Images should be compressed and served in modern formats (like WebP) to ensure fast loading times without sacrificing quality.

**IV. The "Go Live" Process**

* **Pre-Launch SEO Audit:**
  + **Guideline:** Before the final launch, the SEO team needs access to the staging site to perform a final crawl and audit. This helps catch any broken links, missing tags, or other errors before they go live.
* **Launch Day Checks:**
  + **Guideline:** Immediately after launch, the SEO team will need to:
    - Implement and test any necessary 301 redirects to ensure they are working correctly.
    - Run a crawl of the new live site to check for 404 errors (broken pages) and other issues.
    - Submit the new XML sitemap to Google Search Console.
